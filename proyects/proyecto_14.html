Identificacin de Operadores Ineficaces en CallMeMaybe

Introduccin

CallMeMaybe es un servicio de telefona virtual que permite a empresas manejar grandes volmenes de llamadas entrantes y salientes entre sus operadores. Este proyecto tiene como objetivo identificar a los operadores menos eficaces, para facilitar la toma de decisiones por parte de los supervisores.

Un operador se considera ineficaz si cumple con una o ms de las siguientes condiciones:

  • Tiene un gran nmero de llamadas perdidas (especialmente internas y externas entrantes).
  • Registra un tiempo de espera prolongado en llamadas entrantes.
  • Si realiza llamadas salientes, muestra una baja cantidad de llamadas efectivas.

Objetivos

  1. Realizar un anlisis exploratorio de datos (EDA).
  2. Identificar operadores ineficaces segn los criterios establecidos.
  3. Validar hiptesis mediante pruebas estadsticas.
  4. Construir dashboards informativos e interactivos.
  5. Entregar una presentacin con las conclusiones del anlisis.

Fuentes de datos

  • telecom_dataset_us.csv: contiene los registros de llamadas (entrantes, salientes e internas), duracin, operador asignado y si fueron atendidas o perdidas.
  • telecom_clients_us.csv: incluye los planes tarifarios y fechas de registro de los clientes.

En las siguientes secciones se explorarn los datos, se limpiarn si es necesario y se proceder al anlisis detallado.

4. Llamadas perdidas y tiempos de espera

Uno de los indicadores ms importantes de ineficacia operativa es el nmero de llamadas entrantes perdidas, es decir, aquellas en las que el cliente llama y no recibe respuesta.

Otro factor es el tiempo de espera: la diferencia entre total_call_duration y call_duration, que representa el tiempo que el cliente esper antes de ser atendido (o no ser atendido).

En este paso:

  1. Se calcula el nmero de llamadas perdidas por operador.
  2. Se estima el tiempo promedio de espera por operador en llamadas atendidas.
  3. Se identificarn operadores con mayor nmero de llamadas perdidas y mayor tiempo de espera.
# Crear columna de tiempo de espera
df_calls['wait_time'] = df_calls['total_call_duration'] - df_calls['call_duration']

# Filtrar solo llamadas entrantes con operador asignado
df_incoming = df_calls[(df_calls['direction'] == 'in') & (df_calls['operator_id'].notnull())]

# Agrupar por operador para calcular llamadas perdidas y tiempo de espera promedio
summary_operators = df_incoming.groupby('operator_id').agg(
    missed_calls=('is_missed_call', 'sum'),

)

# Crear columna de porcentaje de llamadas perdidas

6. Ranking combinado de operadores ineficaces

Para generar una clasificacin de operadores con bajo rendimiento, se combinaron tres indicadores clave:

  1. Tasa de llamadas perdidas (missed_call_rate)
  2. Tiempo promedio de espera (avg_wait_time)
  3. Volumen de llamadas salientes (total_out_calls)

Cada mtrica fue normalizada (min-max scaling) para permitir la comparacin en una misma escala. Posteriormente, se calcul un ndice de ineficacia ponderado como el promedio de las tres mtricas normalizadas.

Este ndice final permite detectar a los operadores menos eficaces considerando su rendimiento global.

from sklearn.preprocessing import MinMaxScaler

8. Conclusiones del anlisis

El anlisis de desempeo de operadores en CallMeMaybe permiti identificar de forma objetiva los factores que indican ineficacia operativa:

  • Llamadas perdidas: algunos operadores superan el 30% de llamadas entrantes perdidas, un indicador grave de mal servicio.
  • Tiempo de espera: se detectaron operadores con esperas promedio superiores a 8 minutos, incluso cuando las llamadas fueron atendidas.
  • Baja actividad saliente: ciertos operadores apenas realizaron una llamada saliente en todo el periodo, muchas con duracin cero.

Con base en estos tres criterios, se construy un ndice combinado de ineficacia, que permiti generar un ranking claro de los operadores que requieren atencin prioritaria.

Este estudio puede servir como base para:

  • Redisear la distribucin de llamadas entre operadores.
  • Implementar capacitacin focalizada.
  • Ajustar metas individuales de rendimiento.

Nota: El ndice de ineficacia es totalmente ajustable. Despus de la presentacin, se pueden modificar los pesos de cada componente (tasa de llamadas perdidas, tiempo de espera, llamadas salientes) para reflejar con mayor precisin las prioridades estratgicas del rea operativa.
